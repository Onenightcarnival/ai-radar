## Claude Blog

### [The evolution of agentic surfaces: building with Claude Managed Agents](https://claude.com/blog/building-with-claude-managed-agents)
- 日期：2026-06-10
- 摘要：Claude Blog 解释 Claude Managed Agents 的定位：当团队从单轮 API 调用走向可长期运行的 agent 时，生产环境需要托管运行、会话管理、文件系统、执行隔离、凭证管理和可观测性。文章回顾了 Claude API、Claude Code、Claude Agent SDK 到 Managed Agents 的演进，并说明 Managed Agents 通过把 agent harness 与 sandbox 执行环境解耦，让 Claude 可以先开始推理，凭证保留在 vault 和代理层，session 则记录模型调用、工具调用和结果，支持恢复、追踪和调试。
- 关注点：这篇文章把 Managed Agents 的价值说得更完整：不是只提供一个 agent API，而是把长期任务所需的基础设施一起托管。对开发者团队来说，关键点包括凭证不直接暴露给 sandbox、降低容器启动带来的首 token 延迟、持久化 session、内置可观测性、Memory/Dreaming、self-hosted sandboxes 和 MCP tunnels。它也延续了 Anthropic 近期对 Claude Code、Agent SDK 和 Managed Agents 的同一条工程叙事：模型能力上升后，差异化会更多落在上下文管理、领域经验和生产级 harness 上。
