### Building effective human-agent teams
- 链接：[https://claude.com/blog/building-effective-human-agent-teams](https://claude.com/blog/building-effective-human-agent-teams)
- 发布时间：2026-06-24
- 类型：Category
- 摘要：Claude Blog 以 Anthropic 内部实践为例，说明团队与 AI agent 的协作正在从单人聊天窗口转向多人协作空间。文章把 Claude Tag、Claude Code agent teams 等场景称为“multiplayer agents”，重点讨论 agent 在 Slack、代码、文档、会议记录等共享上下文中参与团队工作的条件：公开可检索的信息、清晰的角色和工具权限、团队级 north star，以及通过评审、验证器 agent 和复盘逐步扩大自治范围。
- 关注点：这篇文章更像 Claude Tag / Claude Code agent teams 的组织实践手册，核心信号是 Anthropic 正把 agent 产品从个人效率工具推向团队协作系统，强调文档透明度、角色分工、验证机制和人类注意力管理。

### Agent identity in Claude Tag: a new access model for autonomous, team-wide AI
- 链接：[https://claude.com/blog/agent-identity-access-model](https://claude.com/blog/agent-identity-access-model)
- 发布时间：2026-06-24
- 类型：Category
- 摘要：Claude Blog 解释 Claude Tag 的 agent identity 访问模型：在共享频道中，Claude 不再代表某个具体用户行动，而是通过管理员配置的独立身份访问 Slack、GitHub、数据仓库、连接器、skills、plugins 和频道级 instructions。文章强调权限从“这个用户能做什么”转向“这个 agent 在这个工作区或频道隔离区能做什么”，并说明公开频道、私密频道、DM、RBAC、审计日志和网络边界的差异。
- 关注点：这是 Claude Tag 面向企业部署的关键安全与权限设计。它把多人 agent 协作中的权限、审计、最小权限和工具连接问题产品化，也解释了为什么团队级 agent 不能简单沿用个人账号授权模型。
