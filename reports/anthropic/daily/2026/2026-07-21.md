### How Datadog built a “universal machine tool” for Claude Code
- 链接：[https://claude.com/blog/how-datadog-built-a-universal-machine-tool-for-claude-code](https://claude.com/blog/how-datadog-built-a-universal-machine-tool-for-claude-code)
- 发布时间：2026-07-21
- 类型：Claude Code / Enterprise AI
- 摘要：Claude Blog 介绍 Datadog 如何用 Claude Code 推进面向 agentic software 的 Temper 框架。Datadog 的思路不是让 agent 直接持续生成任意应用代码，而是让 agent 生成规格说明，由一个确定性的 kernel 对行为、数据合约和授权进行验证，再部署可运行系统，以缩小“生成得很快但验证跟不上”的缺口。
- 关注点：这篇把 Claude Code 的价值从代码生成推进到工程体系设计：agent 可以承担更长链路的构建任务，但企业真正需要投资的是可验证、可复用、可审计的运行与验证层，而不是单纯提高代码产出吞吐。

### How Anthropic secures its AI-native software development lifecycle
- 链接：[https://claude.com/blog/how-anthropic-secures-its-ai-native-software-development-lifecycle](https://claude.com/blog/how-anthropic-secures-its-ai-native-software-development-lifecycle)
- 发布时间：2026-07-21
- 类型：Claude Code / Enterprise AI / Agents
- 摘要：Anthropic Deputy CISO Jason Clinton 说明 Anthropic 如何保护由 Claude 深度参与的软件开发生命周期。文章称 Anthropic 当前合入代码中约 80% 由 AI authoring，安全团队因此把项目安全评审、组织知识索引、CLAUDE.md 与 skills、CI/测试、AI-powered DAST、告警分诊和代码迁移纳入新的 SDLC 控制面，并强调 human approval、单用途 agent 身份和最小权限边界。
- 关注点：这是 Anthropic 对“AI-native engineering”安全治理的官方实践披露。对使用 Claude Code 或内部 coding agent 的团队来说，重点是把安全要求嵌入 agent 的上下文、指令和权限系统，同时用独立验证、边界隔离和人工最终责任来控制 blast radius。
