### Project Fetch: Phase two
- 链接：[https://www.anthropic.com/research/project-fetch-phase-two](https://www.anthropic.com/research/project-fetch-phase-two)
- 发布时间：2026-06-18
- 类型：Frontier Red Team 与模型能力评估
- 摘要：Anthropic 复测 2025 年 Project Fetch 机器人实验，让 Claude Opus 4.7 通过 Claude Code 独立完成连接机器狗传感器、编写控制程序、检测目标等任务。官方称，在参与者曾完成的任务上，Opus 4.7 至少快 10 倍；在四个双方都完成的任务上，平均比无 Claude 团队快 37 倍以上、比使用 Claude 的人类团队快 18 倍以上。但模型仍没有可靠完成精细推动球体返回起点这类闭环物理控制任务。
- 关注点：这篇文章把 agentic coding 的进展延伸到物理工具使用场景。重点信号不是“机器人问题已解决”，而是模型从辅助人类转向更大比例自主完成工具接入、代码生成和调试；同时，精细低层控制仍是明确边界。

### Claude Code now supports artifacts
- 链接：[https://claude.com/blog/artifacts-in-claude-code](https://claude.com/blog/artifacts-in-claude-code)
- 发布时间：2026-06-18
- 类型：Claude Code 产品能力
- 摘要：Claude Code 新增 artifacts beta，可把当前会话进展生成可交互、可分享的网页，例如 PR walkthrough、系统说明、仪表盘、发布清单或事故排查页。Artifacts 会利用会话中的代码库、连接器和对话上下文，支持同一链接下的版本更新；默认仅作者可见，可分享给团队或组织，且不能公开访问。
- 关注点：这把 Claude Code 的输出从终端对话推进到团队可消费的工作页面。对工程团队来说，它主要解决 agent 工作过程的可见性和协作交接问题，尤其适合调试、审查、合规、发布和跨团队状态同步。

### Centrally manage authorization for MCP connectors
- 链接：[https://claude.com/blog/enterprise-managed-auth](https://claude.com/blog/enterprise-managed-auth)
- 发布时间：2026-06-18
- 类型：Claude Enterprise 与 MCP 治理
- 摘要：Claude 推出 MCP connectors 的 enterprise-managed authorization beta，管理员可通过身份提供商为组织集中配置连接器授权，首发支持 Okta。用户首次登录 Claude 时会自动获得其身份组和角色允许的连接器访问权限；该能力基于 MCP Enterprise-Managed Authorization 扩展，覆盖 Claude chat、Claude Code 和 Cowork。
- 关注点：这是一条企业 MCP 落地信号。重点在于把连接器访问纳入现有 IdP、组和角色治理，减少逐个用户 OAuth 授权和个人账号误连风险；Asana、Atlassian、Canva、Figma、Granola、Linear、Supabase 等 MCP provider 已支持或将支持该模式。
