### Working with Claude Fable 5 in Claude Cowork
- 链接：[https://claude.com/blog/working-with-claude-fable-5-in-claude-cowork](https://claude.com/blog/working-with-claude-fable-5-in-claude-cowork)
- 发布时间：2026-07-16
- 类型：Product / Claude Cowork / Claude Fable 5
- 摘要：Claude Blog 说明如何在 Claude Cowork 中使用 Claude Fable 5。文章强调 Fable 5 适合复杂、模糊、跨多步的知识工作，能结合用户提供的文件、工具和 skills 规划任务、拆分子任务并在执行中检查结果；同时建议按任务复杂度选择 effort 设置，并说明在网络安全、生物和化学相关请求上会由独立分类器触发安全处理，必要时切换到 Claude Opus 4.8。
- 关注点：这篇文章把 Claude Fable 5 定位为 Claude Cowork 中面向长任务和高判断成本工作的模型，而不是日常快速任务默认选项。对产品和团队使用者来说，重点是如何把目标、约束、上下文和可验收结果交给 Cowork，让模型承担更完整的端到端知识工作。

### How Anthropic runs large-scale code migrations with Claude Code
- 链接：[https://claude.com/blog/ai-code-migration](https://claude.com/blog/ai-code-migration)
- 发布时间：2026-07-16
- 类型：Claude Code / Engineering / Enterprise AI
- 摘要：Claude Blog 发布 Anthropic 使用 Claude Code 执行大规模代码迁移的方法。文章以 Bun 从 Zig 迁移到 Rust、以及内部 Python 代码库迁移到 TypeScript 为例，介绍 Claude Fable 5、Claude Opus 4.8 和动态工作流如何支持并行 agent、阶段门、对抗式 review、测试和输出一致性校验等流程；其中 Bun 迁移在两周内生成约百万行代码，合并前通过既有 CI 测试，合并后发现的 19 个回归也已修复。
- 关注点：这是一篇面向工程团队的 Claude Code 实战方法论，核心不是简单让模型“改代码”，而是把迁移拆成可并行、可验证、可复盘的循环。对有遗留语言、性能瓶颈或长期迁移债务的团队，文章给出了用 AI agent 降低迁移成本和风险的官方范式。
