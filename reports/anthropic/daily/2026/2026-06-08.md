### [Paving the way for agents in biology](https://www.anthropic.com/research/agents-in-biology)
- 日期：2026-06-08
- 摘要：Anthropic Research 介绍了一项关于生物学 agent 基础设施的研究。团队让 Claude、Biomni、Edison Analysis、GPT 等科学研究 agent 从 NCBI Virus 检索病毒序列数据，发现即使较强模型也难以稳定达到可靠数据集构建所需的准确性；当加入 `gget virus` 这样的确定性检索层后，agent 的准确率和可复现性显著提升。
- 关注点：这篇文章的核心不是简单展示模型能力，而是指出科学 agent 的瓶颈常常在数据库、接口、元数据和可验证执行层。对 AI for science、生命科学工具和 agent harness 设计来说，值得关注的是 Anthropic 明确强调“让生物数据基础设施适配 agent”，并把可靠检索、审计日志和标准化输出视为科学工作流落地的关键条件。

## Claude Blog

### [Building intelligent apps for Apple platforms with Claude in the Foundation Models framework](https://claude.com/blog/claude-for-foundation-models)
- 日期：2026-06-08
- 摘要：Claude Blog 宣布通过新的 Swift package 支持 Apple 的 Foundation Models framework，让 Apple 开发者可以在 Swift 应用中把本地模型任务和 Claude API 调用衔接起来。文章说明，开发者可先用 Apple on-device 模型处理快速、本地、结构化任务，再在需要多步推理、代码生成、联网检索或代码执行时交给 Claude。
- 关注点：这是 Claude 进入 Apple 原生应用开发链路的产品信号。它强调的不是替代端侧模型，而是在同一用户体验中按任务复杂度选择合适模型，并通过 typed Swift values、streaming、tool calls 和 structured responses 降低开发者接入 Claude 的成本。

### [Observability for developers building connectors](https://claude.com/blog/observability-for-developers-building-connectors)
- 日期：2026-06-08
- 摘要：Claude Blog 发布连接器开发者可观测性更新。已发布到 Claude directory 的连接器现在可以通过 dashboard 查看跨 Claude 产品表面的表现，包括活跃用户、工具调用量、目录排名、健康分、错误率、延迟和按工具拆分的错误情况；开发者也可以直接在 Claude 中提交 MCP server 到目录。
- 关注点：这篇内容说明 Anthropic 正在把 MCP 连接器生态从“能接入”推进到“能运营、能诊断、能分发”。对企业和工具开发者来说，关键变化是连接器不再只是协议实现，还需要围绕 Claude、Claude Code、Cowork 等产品表面观察真实使用、排查失败并优化质量。
