## Anthropic Research

### Coding agents in the social sciences
- 日期：2026-05-27
- 链接：https://www.anthropic.com/research/coding-agents-social-sciences
- 摘要：Anthropic 发布一项面向 1,260 名社会科学研究者的调查，观察 AI 和 coding agents 在科研中的采用情况。调查显示，81% 的受访者尝试过用 AI chatbot 辅助研究，但只有 20% 定期使用 Claude Code、Codex、Cursor 等 coding agents；采用率在学科、职业阶段、性别化姓名和学校层级之间存在明显差异。文章还指出，coding agent 用户在工作论文和基金申请等早期产出上更多，但这只是描述性相关，不能直接解释为因果关系。
- 关注点：这篇研究把 Claude Code 等 agentic coding 工具放进社会科学研究流程中讨论，重点不是“AI 是否能写论文”，而是 coding agents 是否会改变数据分析、项目启动和研究产出节奏。对关注 AI 影响科研生产率、公平性和知识生产结构的读者来说，这是 Anthropic Economic Research 线索中值得跟踪的一篇。

## Claude Blog

### Using LLMs to secure source code
- 日期：2026-05-27
- 链接：https://claude.com/blog/using-llms-to-secure-source-code
- 摘要：Claude Blog 总结了用 Claude Opus 做源码安全工作的实践方法：先建立 threat model 和 sandbox，再让模型发现漏洞，并通过独立验证、分级去重和补丁验证形成闭环。文章强调，当前模型已经让漏洞发现更容易并行化，但真正的瓶颈转移到验证、triage 和修复；官方还提供了配套 repo，包含交互式 workflow skills 和自动扫描 demo harness。
- 关注点：这是今天最值得安全与平台团队关注的实践文章。它把 Claude 从“辅助找 bug”推进到可操作的安全工程流程，强调隔离环境、可复现验证、独立 verifier、最小补丁和回归测试，避免把模型发现直接等同于可利用漏洞。

### How CodeRabbit used Claude to build an agent orchestration system
- 日期：2026-05-27
- 链接：https://claude.com/blog/how-coderabbit-used-claude-to-build-an-agent-orchestration-system
- 摘要：Claude Blog 介绍 CodeRabbit 如何用 Claude Platform 和 Claude Code 构建 agent orchestration layer。CodeRabbit 观察到，AI 生成代码常见失败不是编译不过，而是“代码能跑、测试能过，却没有解决真实需求”。为降低这类偏差，他们在代码生成前加入规划阶段，用 Claude 分析需求、显式化假设、产出可评审的 PRD，再交给 Claude Code 做更细的实现计划。
- 关注点：这篇文章给出一个清晰的 agentic coding 组织模式：先用编排层把隐含需求变成可讨论的计划，再让 coding agent 执行。对正在把 Claude Code 接入团队开发流程的组织来说，关键启发是把 plan quality 作为质量门，并用 eval harness 衡量计划粒度、代码结果和 token 成本，而不是只盯最终代码生成速度。

### Zero Trust for AI agents
- 日期：2026-05-27
- 链接：https://claude.com/blog/zero-trust-for-ai-agents
- 摘要：Claude Blog 发布一篇 Enterprise AI 方向内容，讨论 AI agents 进入企业系统后需要以 Zero Trust 思路管理访问、行为和审计边界。官方列表将其归入企业 AI 内容，核心议题是 agent 不应因为自动化能力增强就获得不受约束的系统权限，而应围绕身份、资源访问、操作范围和可追踪性建立控制。
- 关注点：这条内容和 Anthropic Engineering 近期关于 containment、Claude Code 权限、Managed Agents 沙箱等文章形成呼应。对企业落地 Claude agents 的团队来说，重点是把 agent 当作需要身份、最小权限、网络与工具边界、日志审计的执行主体，而不是普通聊天窗口。
