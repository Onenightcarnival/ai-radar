## Anthropic News

### Introducing the Services Track and Partner Hub of the Claude Partner Network
- 日期：2026-06-03
- 链接：https://www.anthropic.com/news/services-track-partner-hub
- 摘要：Anthropic 宣布为 Claude Partner Network 增加 Services Track 和 Claude Partner Hub。Services Track 用 Select、Preferred、Global Premier 三个层级衡量服务伙伴的 Claude 实践深度，指标包括活跃认证人员、过去 12 个月生产部署客户数量、公开客户案例和联合业务计划等。Claude Partner Hub 则提供伙伴自身达标情况、客户可查询的公开目录，以及一个可连接 Claude 的 MCP connector，方便伙伴直接询问认证、交易注册和层级进展。
- 关注点：这是企业生态和渠道信号。Anthropic 不只是扩大伙伴名单，而是在用可验证的认证、部署和客户背书约束服务伙伴质量。对企业客户来说，Partner Hub 把“谁真正做过 Claude 生产落地”变成更可比较的信息；对服务商来说，MCP connector 也说明 Anthropic 希望把伙伴管理流程直接带入 Claude 工作流。

### What we learned mapping a year’s worth of AI-enabled cyber threats
- 日期：2026-06-03
- 链接：https://www.anthropic.com/news/AI-enabled-cyber-threats-mitre-attack
- 摘要：Anthropic 基于 2025 年 3 月至 2026 年 3 月因恶意网络活动被封禁、且有足够细节可分析的 832 个账号，将其活动映射到 MITRE ATT&CK 框架。文章指出，恶意行为者越来越多把 AI 用在攻击链后段和更复杂的操作中，例如横向移动、账号发现和权限提升；传统上用技术数量、工具界面或攻击者熟练度来判断风险的方法正在变弱。文章还认为，MITRE ATT&CK 尚未充分覆盖 AI agent 串联攻击阶段、实时决策和低人工介入执行等新行为。
- 关注点：这是安全研究和政策信号。Anthropic 将封禁数据、Verizon DBIR 合作、Frontier Red Team 分析和 Project Glasswing 放在同一条防御叙事里，强调 frontier model 改变了攻防两侧的能力边界。值得关注的是，文章把“攻击者搭建模型脚手架的能力”视为更持久的风险区分因素，这与 Claude Code、agent orchestration 和安全护栏设计直接相关。

## Claude Blog

### How Anthropic enables self-service data analytics with Claude
- 日期：2026-06-03
- 链接：https://claude.com/blog/how-anthropic-enables-self-service-data-analytics-with-claude
- 摘要：Claude Blog 介绍 Anthropic 内部如何用 Claude 支持自助式业务分析。文章称，Anthropic 约 95% 的业务分析查询已由 Claude 自动化处理，整体准确率约为 95%。文章把准确性问题归结为上下文和验证问题，而不是单纯的 SQL 生成问题，并拆解了概念与实体歧义、数据过期、检索失败三类主要错误来源。对应方案包括更强的数据基础、语义层、数据血缘、业务上下文、Claude Code skills、离线 eval、在线验证和纠错采集。
- 关注点：这是企业数据 agent 的工程实践信号。文章的重点不是让 Claude “能写 SQL”，而是把数据治理、语义层、技能文档、CI 和 eval 组合成可维护的 agentic analytics stack。对数据团队来说，最有价值的经验是：提高准确率往往需要缩小 agent 的选择空间、维护可检索的业务语义，并用 adversarial review 与 provenance footer 管住沉默错误。

### Lessons from building Claude Code: How we use skills
- 日期：2026-06-03
- 链接：https://claude.com/blog/lessons-from-building-claude-code-how-we-use-skills
- 摘要：Claude Blog 总结 Anthropic 内部在 Claude Code 中使用数百个 skills 的经验。文章强调，skills 不是“只有 markdown 的提示词”，而是可包含说明、脚本、资源和配置的文件夹，用来帮助 agent 更准确、更高效地完成特定任务。文章按用途归纳了内部 skills 的类型，并讨论 API/库参考、产品验证、工作流封装、领域知识和分发维护等设计取舍。
- 关注点：这是 Claude Code 的核心工程信号。Anthropic 正把 skills 定位为 Claude Code 的重要扩展点，用来沉淀组织知识、验证流程和工具使用方式。对开发团队来说，文章提示 skills 的价值不在于把提示写长，而在于把“什么时候读什么、怎么验证、哪些脚本可复用”做成 agent 能稳定发现和执行的结构。

### Best practices for getting started with Claude Cowork
- 日期：2026-06-03
- 链接：https://claude.com/blog/best-practices-for-getting-started-with-claude-cowork
- 摘要：Claude Blog 用 Anthropic 增长营销负责人 Austin Lau 的实践说明如何开始使用 Claude Cowork。文章把 Claude Chat、Claude Cowork 和 Claude Code 区分为不同工作空间：Chat 更适合问答、脑暴和短交流；Claude Cowork 适合生成可交付成果、处理多步骤任务、跨文件或跨应用工作；Claude Code 则面向开发者。文章还给出从选择任务、连接工作文件夹和应用，到把交付物交给 Claude Cowork 执行的入门路径。
- 关注点：这是产品采用和工作流教育信号。Anthropic 正在把 Claude Cowork 描述为非技术知识工作者可用的 agent workspace，而不是开发者专属工具的变体。值得关注的是，文章明确说 Claude Cowork 与 Claude Code 使用同一底层 engine，这说明 Anthropic 可能会把面向开发者沉淀出的长任务、文件操作和多步骤执行能力迁移到更广泛的企业岗位。

### Running an AI-native engineering org
- 日期：2026-06-03
- 链接：https://claude.com/blog/running-an-ai-native-engineering-org
- 摘要：Claude Blog 整理 Code w/ Claude SF 2026 中 Claude Code 和 Claude Cowork 工程负责人 Fiona Fung 的分享，讨论当 agentic coding 成为默认工作方式后，工程组织流程如何变化。文章认为，写代码、写测试和重构不再是主要瓶颈，验证、代码审查和安全开始成为新的约束。Claude Code 团队因此把规划从长周期路线图转向更即时的原型和 PR 讨论，把上下文查询从“找作者”转向“让 Claude 汇总事实”，并在代码审查中让 Claude 处理风格、lint、常规反馈和测试，保留人类对专业判断、信任边界、安全和产品品味的把关。
- 关注点：这是 agentic engineering 组织实践信号。文章把 Claude Code 的影响从个人效率扩展到团队流程：规划、上下文收集、评审、安全和产品判断都需要重新分工。对技术管理者来说，重点不是“人还要不要审代码”，而是哪些环节可以由 Claude 先做、哪些信任边界必须由领域专家保留，以及这些边界会如何随模型能力继续变化。
