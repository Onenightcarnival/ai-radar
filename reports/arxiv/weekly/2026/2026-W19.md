# arXiv 论文周报 2026-W19

这一期按 2026-W19 归档，目标工作周是 2026-05-04 至 2026-05-08。仓库里目前只有 2026-05-04、2026-05-05、2026-05-06、2026-05-07 四期 arXiv 日报，所以这份周报只基于现有日报做二次提炼，不补搜、不读缓存，也不把缺失日期硬写成完整五天趋势。和上一期相比，本周最清楚的变化是：Agent 安全从单点 prompt injection 扩展到运行时、记忆、终止条件、工具调用和评测基础设施；RAG 从“能不能检索到”转向“会不会泄露、被投毒、被文本偏置或被架构拖累”；代码 Agent 评测也更贴近生产代码库、测试演化和工程约束。

## 本周趋势概览

本周反复出现的第一条主线是 Agent 安全和运行时治理。MAGE、ARGUS、DTap、AgentTrust、LoopTrap、MANTRA、SafeHarbor、PragLocker、TurnGate 等论文都不再只讨论模型是否拒答，而是在看长程轨迹、工具调用、记忆写入、环境注入、终止条件、合规手册和 prompt 资产保护。这个方向的共同判断很明确：Agent 上线后的风险来自系统交互，不只来自单轮输入。

第二条主线是 Agent 记忆和长程任务训练。MemAudit、MemSAD、MEMTIER、True Memory、BeliefMem、STALE、MemReranker、SkillOS、SkillRet、BEACON 等论文从不同角度处理“记什么、怎么找、旧记忆何时失效、技能库如何演化、长程奖励怎么分配”。这不是单日偶发现象，而是连续四期日报都在出现的主题。

第三条主线是 RAG 和搜索系统进入安全、结构化和预算控制阶段。Architecture Matters、LeakDojo、AgenticRAG、LatentRAG、TGS-RAG、SCOUT、LongSeeker、OpenSeeker-v2、OpenSearch-VL、BRIGHT-Pro、复杂集合检索复现等论文都在说明：RAG 的问题已经从简单向量召回，扩展到架构鲁棒性、知识库泄露、图文证据冲突、多轮检索控制、长文档主动取证和推理密集检索评测。

第四条主线是代码 Agent 的评测更接近真实工程。ProgramBench、SWE-WebDevBench、MOSAIC-Bench、TEBench、Constraint Decay、BUILD-AND-FIND、KVerus、Root-Cause-Driven Automated Vulnerability Repair、ARISE、AuditRepairBench 等论文反复强调同一件事：能生成代码不等于能维护项目，能修 bug 不等于修了根因，能通过功能测试不等于满足架构、安全、测试和可读性约束。

医疗和科学 Agent 也明显升温，但本周更像“场景化基础设施”而不是单一模型突破。PhysicianBench、BioMedArena、BioResearcher、BioTool、NeuroAgent、CuraView、MedFabric/EtHER、SaFE-Scale、BioMedArena 等论文都在做数据、工具、评测和证据链。这个方向值得跟，但要警惕 LLM judge、私有工具链和小样本评测带来的外推风险。

## 高频主题归类

**Agent 安全从提示词防御转向运行时治理**

本周安全论文覆盖面很宽：长程威胁用 shadow memory 管，context-aware injection 用 ARGUS 防，真实应用沙盒用 DTap 评测，工具调用用 AgentTrust 拦截，终止逻辑用 LoopTrap 攻击，合规轨迹用 MANTRA 生成并用 SMT 检查，prompt 资产用 PragLocker 保护，多轮恶意意图用 TurnGate 盯回复闭合点。连续信号很强，且多数论文都把风险放在 Agent 与环境、工具、记忆和用户的交互里。

**Agent 记忆成为独立系统问题**

记忆相关论文不是在重复“加一个向量库”。MemAudit 把记忆写入质量拆出来评估，MemSAD 看长期记忆投毒，MEMTIER 做分层记忆，True Memory 强调原始事件和检索管线，BeliefMem 保留不确定性，STALE 看旧记忆失效，MemReranker 补推理型重排。它们共同说明长期 Agent 的难点在记忆生命周期，而不是单次检索技巧。

**RAG 正在被安全、架构和证据链重写**

RAG 本周有三类反复信号。第一类是安全：知识库投毒、私有文本泄露、faithfulness 和泄露风险之间的张力。第二类是架构：vanilla、agentic、多智能体、递归语言模型、GraphRAG、文本-图互证会产生不同鲁棒性。第三类是控制：SCOUT、LongSeeker、LatentRAG、AgenticRAG 都在压缩多轮检索的成本，或者把证据查找变成可控动作。

**代码 Agent 评测从 pass rate 转向可维护性和约束**

ProgramBench 看从零重建程序，SWE-WebDevBench 看完整业务应用交付，TEBench 看测试演化，Constraint Decay 看后端约束退化，MOSAIC-Bench 看多票据组合漏洞，BUILD-AND-FIND 看后续 Agent 能否理解生成代码库。这个方向已经从“模型会不会写代码”转到“生成物能不能被团队和后续 Agent 维护”。

**医疗和科学 Agent 更依赖工具链与评测基建**

PhysicianBench、BioMedArena、BioResearcher、BioTool、NeuroAgent、CuraView、SaFE-Scale、MedFabric/EtHER 都在强调证据、工具、流程、场景和安全评估。值得注意的是，很多结果仍依赖 LLM judge、小规模样本或自建 benchmark，所以本周更适合看成基础设施密集出现，而不是某个医学 Agent 已经解决真实临床工作。

**多模态从单任务能力转向长程、空间和跨模态冲突**

OpenSearch-VL、SFI-Bench、VTAgent、Event-Causal RAG、ReasonAudio、MSEB、SWE-WebDevBench 里的视觉评测，以及空间智能、室内场景、机器人扫描参数等论文，都在处理多模态模型的证据选择、空间关系、长视频记忆、音频推理和跨模态偏置。这个方向有连续性，但应用差异很大，暂时不宜合成一个过宽的“多模态爆发”判断。

## 与上一期周报相比的变化

上一期 2026-W18 主要基于单日样本，主线是 Agent 工程化、RAG/结构化检索、代码生成真实评测和多模态长程决策。本期虽然日报仍不完整，但样本从单日增加到四期，信号更稳定。

最明显的变化是 Agent 安全的颗粒度变细了。上一期更像是在讨论 skill、workflow、GUI 观察和 RAG 知识库治理；本期已经扩展到运行时拦截、工具调用审计、记忆投毒、终止攻击、合规轨迹、prompt 资产保护、部署上下文差异和多轮恶意意图闭合点。

RAG 也从“结构化检索和去噪”进一步走向“架构选择和风险控制”。本期反复出现投毒、泄露、GraphRAG 互证、长文档主动取证、搜索 Agent 预算控制和多模态 RAG 文本偏置，说明 RAG 已经不只是检索质量问题，而是系统可靠性问题。

代码方向的变化是评测更像真实工程流程。上一期已有科研复现、偏见、能耗和复杂需求；本期进一步看到项目重建、完整 Web 应用、测试演化、架构约束退化、组合漏洞和后续可理解性。这个变化值得继续跟。

医疗和生物医学 Agent 是本期新增更强的方向。上一期医疗更多是 RAG 隐私和单点应用，本期出现 PhysicianBench、BioMedArena、BioResearcher、BioTool、NeuroAgent、CuraView、SaFE-Scale 等一组工具链和评测基础设施论文，连续性明显增强。

## 下周值得继续跟踪的问题

- Agent 安全是否会形成运行时标准层：工具调用前拦截、记忆写入审计、终止检测、合规 trace check 和人工接管是否能合在一起。
- 长期记忆方向是否会从 benchmark 走向产品约束：旧事实过期、来源可信度、权限边界、写入预算和重排器校准都还需要真实系统验证。
- RAG 安全是否会把“faithfulness”重新定义，因为忠实复述和私有知识泄露之间可能存在冲突。
- 代码 Agent 评测是否会持续加入测试演化、架构约束、组合漏洞、可维护性和后续 Agent 理解成本，而不是只看一次性修复率。
- 医疗 Deep Research Agent 的工具和评测能否减少 LLM judge 依赖，并证明在真实临床或科研流程里有可复现收益。
- 搜索 Agent 的训练是否会继续沿着高难度轨迹、上下文编排、预算控制和多模态检索这几条线收敛。
